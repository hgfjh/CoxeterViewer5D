# Release Notes

Release notes live here before they are copied into a GitHub release, package
description, or demo handout. Keep them factual and separate mathematical claims
from viewer, tooling, and packaging changes.

For each release, create a file named `vX.Y.Z.md` from
`template.md`. Record:

- User-visible app changes.
- Mathematical data or certificate changes.
- External-tool and diagnostics changes.
- Web and desktop packaging status.
- Signing and updater status.
- Validation commands and known blockers.

Unsigned local desktop bundles are acceptable for development builds. A public
desktop release should say whether platform code signing and updater signing
were configured, skipped, or blocked.
